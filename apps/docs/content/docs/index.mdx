**better-s3** is a customizable, AWS SDK-first layer for S3-powered app flows.

You still create and configure your own `S3Client`, point it at any S3-compatible provider, and keep full control over credentials, endpoints, buckets, and request behavior.

better-s3 adds a **recommended layer** on top: typed server handlers, React hooks, and optional UI components for the flows most apps end up rebuilding around S3.

It is **presigned first**. Your server signs requests and enforces rules, while file bytes move directly between the client and storage.

Works with **AWS S3**, **Cloudflare R2**, **MinIO**, **DigitalOcean Spaces**, **Backblaze B2**, and other S3-compatible providers.

### How it works

1. Your app exposes typed server handlers for presign, confirm, download, delete, and multipart actions.
2. Hooks or UI components call those handlers from the client.
3. File bytes move directly between the browser and your S3 provider.
4. Your app stays in sync through validation, progress state, and lifecycle hooks.

## Why better-s3?

- **AWS SDK first:** keep the native client, provider config, and low-level control you already know.
- **A recommended architecture:** start from a complete flow instead of stitching S3 pieces together yourself.
- **Built for app flows:** uploads, downloads, deletes, multipart, validation, and lifecycle hooks in one stack.
- **Proxy-free by design:** the server signs requests without becoming a bandwidth bottleneck.
- **shadcn-friendly UI:** the UI package follows shadcn styling and fits naturally into apps already using shadcn/ui.
